# 架构基线

## 固定的一层

除非项目方向发生重大变化，下列部分不轻易变动：

- 产品形态：面向跨厂商编码代理的 CLI-first 编排引擎
- 核心流程：Architect -> Workers -> Reviewer -> Repair Loop -> Final Output
- 契约优先：任务、所有权和验收规则必须结构化
- 本地优先：每次运行都必须在本地留下产物和轨迹
- 优先使用现成积木，再考虑自研

## 迭代的三层

以下三层允许按版本持续演进：

1. 模型层
   选择 provider、路由规则、fallback 策略、成本策略。
2. 工作流层
   提升规划质量、审查质量、修复闭环和 Git 集成。
3. 产品层
   补充编辑器集成、桌面壳、Web 控制台、团队控制和企业治理。

## 现有积木

- `LiteLLM`
  多厂商模型统一网关
- `MCP`
  标准化工具接入层
- `Git`
  工作区隔离、版本对比和合并基础
- `Pydantic`
  结构化任务协议与运行状态
- `Typer` + `Rich`
  CLI 与终端交互体验

## 约束

- 不依赖自由文本聊天作为主协议
- 能用机器校验时，优先机器校验再交给模型审查
- worker 必须有明确边界，避免改同一文件打架
- 每次运行都要可审计、可复盘
